Deduction order

What comes off the gross first. Drag to reorder — this resolves the “wait, were those fees on top or off the top?” arguments before they happen.

Renders only when there’s something to order. Flat with no recoups → this whole section disappears. % of gross and Door get a simplified 3-stage order (no in-cap step).

Recommendation
All three variations share the same backbone: one component, one union data model, sections gated by deal.dealType. That’s the implementation hint from the spec, and it’s what makes the “form can’t send an ambiguous deal” promise enforceable — the required set itself adapts.

Ship A · Worksheet first. The deal-type segmented control at the top of the Structure card drives the rest of the page; locked basis, conditional caps, the Door-only walkout pot, and the conditional deduction order all flow from it. Steal B’s Step 01 fork screen as the very first interaction the first time a venue creates a deal — it makes the choice explicit when the muscle memory isn’t there yet. Reuse C’s live recap as the read-only confirm screen between “ready to send” and the actual send.
